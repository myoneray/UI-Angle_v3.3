Getting started

Download MeteorJS

Start downloading METEOR framework for your platform at https://www.meteor.com/install

Start the application

Run the following command in the meteor project root folder
meteor

Open your browser

If eveything goes fine, your application should be ready at http://localhost:3000/

 

General tips

  • Do not start from scratch, use an existing asset and modify it to learn how it works.
  • Explore the sources for ideas and sample code.
  • Use Firebug or Chrome Developer Tools to find bugs on your website. Using one of those tools will help you to save time analyzing the site and finding elements structure, like classes, id or tags
  • Quick tip: open your site with Chrome, press F12 and go to console tab, reload your page and if something goes wrong you will see your page errors in red text.
  • In case of error messages, someone might have seen it too, so you can try a Google search for a quick fix.

Structure

Structure 

 
We have used the following structure:
Since MeteorJs doesn't have a strict folder structure, feel free to adapt it to your needs.
 
+---client
|   +---angle
|   |   +---js
|   |   |   +---modules
|   |   +---less
|   |       +---app
|   |       +---bootstrap
|   |       +---themes
|   +---lib
|   |   +---js
|   +---vendor
|   +---views
|       +---pages
|       +---partials
+---private
|   +---i18n
+---public
|   +---api
|   +---documentation
|   +---img
|   +---themes
|   +---vendor
+---server
 
client: contains the modules loaded in the client. 
 
  - angle: contains the main files used in the Angularjs project with the same structure for Javascript and Less styles
 
  - lib: contains the script that are loaded before other scripts. Here is the main module declaration
 
  - vendor: contains the vendor files loaded when the application start. Not lazy loaded.
 
  - views: contains the HTML views used in the application
 
private: this folder is only accesible from the server.
 
puiblic: accesible from the client side. It can contain the the main assets like images, fonts, etc
 
  - themes: Contains all predefined themes
  - api: This is folder contains static json files used to simulated response data from an api system.
 
  - vendor: This is folder is special because here is where we place vendor assets that are loaded using the ocLazyLoad module
 
server: contains the script that runs only in the server.
 

 

Layout

Layout and Routes

 
Layout and Routes are managed by AngularJS at the client side using the UI Router module.
The index.html file only contains the necessary markup to start an AngularJS application
 

Index markup

<head>
   <base href="/">
   <meta charset="utf-8">
   <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
   <meta name="description" content="{{app.description}}">
   <meta name="keywords" content="app, responsive, angular, bootstrap, dashboard, admin">
   <title data-ng-bind="::pageTitle()">Angle - Angular Bootstrap Admin Template</title>
</head>

<body data-ng-class="{ 'layout-fixed' : app.layout.isFixed,
                        'aside-collapsed' : app.layout.isCollapsed,
                        'layout-boxed' : app.layout.isBoxed,
                        'layout-fs': app.useFullLayout,
                        'hidden-footer': app.hiddenFooter,
                        'layout-h': app.layout.horizontal,
                        'aside-float': app.layout.isFloat,
                        'offsidebar-open': app.offsidebarOpen,
                        'aside-toggled': app.asideToggled}">

  <div data-preloader></div>
  <div data-ui-view="" data-autoscroll="false" class="wrapper"></div>

</body>

 

Common templates and views

The main layout uses common templates like topnavbar, sidebar, offsidebar and footer.
All common templates can be found under /client/views/partials
All views can be found under /client/views folder.
 

Modules

Modules

 
The application is started using the angular bootstrap method in the file app.module.js.
This files contains also the main module declaration
 
(function() {
    'use strict';

    angular
        .module('angle', [
            'app.core',
            'app.routes',
            'app.sidebar',
            'app.navsearch',
            'app.preloader',
            'app.loadingbar',
            'app.translate',
            'app.settings',
            'app.dashboard',
            'app.icons',
            'app.flatdoc',
            'app.notify',
            'app.bootstrapui',
            'app.elements',
            'app.panels',
            'app.charts',
            'app.forms',
            'app.locale',
            'app.maps',
            'app.pages',
            'app.tables',
            'app.extras',
            'app.mailbox',
            'app.utils'
        ]);

    if (Meteor.isCordova)
      angular.element(document).on("deviceready", onReady);
    else
      angular.element(document).ready(onReady);

    function onReady() {
      angular.bootstrap(document, ['angle']);
    }

})();
 
As you can see, we use angular.bootstrap to manually start the application due to Meteor doesn't allow to define custom attributes for the HTML tag.
 
Modules organization
 
Within folder /client/lib/js lives all modules declaration. They are located under this lib folder so Meteor loads them before any other JS because we need to declare first our modules and then use them. 
 
Any other module declaration must be placed here and the rest of files (config, controllers, etc) should be placed whing the client/angle/js folder with their respective folder name.

Themes

By editing the file /client/angle/less/bootstrap/variables.import.less you have access to all main variables used to build the styles, changes its values to change a color.

Themes selector

The themes selector used in the right sidebar (offsidebar) uses the directives load-css. This directive receives a path to a css stylesheet to be injected into the head section of the page. 

In the demo, we link to the files under the /public/themes folder which contains the predefined theme colors. Then from html we used the following attribute to create an element that when is clicked injects the css in the given path

<input type="radio" name="setting-theme" load-css="/themes/theme-b.css" />

The example shows an input but any clickable element can be used instead
